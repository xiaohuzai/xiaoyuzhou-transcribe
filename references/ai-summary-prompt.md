# AI 总结 Prompt 设计参考

> 适用于小宇宙播客转写流程的 AI 处理阶段（说话人标注 + 错误修正 + 结构化总结）。
> 说话人标注 + 错误修正 + 结构化总结（Part A + Part B）。

## 设计原则

1. **说话人标注**：主播/嘉宾实名，由 LLM 根据对话内容（称呼、语气、话题切换）分配
2. **错误修正**：有把握的错误直接改，不确定标注【疑似误识：原文】
3. **上下文注入**：播客元数据（播放量/时长/嘉宾）作为背景，不搜其他播客
4. **时间戳**：`[MM:SS]`，ASR 输出的 `Chinese` / `English` 行头标记在输出时去掉
5. **Part A 不重写**：format_transcript.py 已经做了说话人+时间戳，LLM 只做小修。完整重写 64KB transcript 跑 8+ 分钟被 SIGPIPE 截断。

## Prompt 模板

```text
你是一个播客内容分析师。请处理以下播客转写内容，生成两部分输出。

【转写内容】
<ASR原始转写文本（已含说话人标注 + [MM:SS] 时间戳）>

【节目元数据】
标题：<TITLE>
时长：<DURATION_MIN> 分钟
播放量：<PLAY_COUNT>
评论数：<COMMENT_COUNT>
描述：<DESCRIPTION>

【输出要求】

## Part A — 带说话人标注的完整转写（修正版，仅修正不重写）

格式：
**[MM:SS] 说话人名**：对话内容

规则：
- 在原 transcript 基础上**只修正关键错误**（人名/术语/说话人归属）
- 不要逐句重复全部 transcript，引用即可
- 说话人优先使用关键词表中的实名
- ASR 对专有名词（人名/公司名/技术术语）的错误：有把握的直接修正，不确定的在括号内标注【疑似误识：原文】
- 语气词和重复内容可精简但不删改原意
- 时间戳使用 [MM:SS] 格式

## Part B — 结构化总结（核心产出）

### 基本信息
- 标题：
- 主播：
- 嘉宾：
- 时长：
- 播放量：

### 完整节目简介
（2-3 段话概括本期核心内容）

### 章节时间线
（每段 [MM:SS] + 一句话摘要，5-8 段）

### 核心要点（按主题分组，3-5 组即可）
### 1. [主题一]
- 要点 + 说话人
### 2. [主题二]
- 要点 + 说话人

### 关键金句（3-5 句，带说话人标注）
> 「金句内容」 —— 说话人名

### 关键问答（3-5 对 Q&A）
**Q1**: 问题？
**A1**: 回答（注明说话人）

请直接输出 Part A 和 Part B 两个 markdown 块，不要开场白或解释。
Part A 用 `## Part A` 开头，Part B 用 `## Part B` 开头。
```

## 错误修正判断规则

| 情况 | 处理方式 |
|------|---------|
| 人名/公司名/技术术语明显错误 | 直接修正 |
| ASR 输出的语言标记行（`Chinese` / `English`） | 直接删除，不出现在最终输出 |
| 不确定的人名/术语 | 标注【疑似误识：原文】 |
| 语气词重复（如"对对对对"） | 精简为"对对"，不改原意 |
| 话题切换点 | 在 Part A 中自然分段，Part B 时间线独立反映 |

## 关键设计：max_tokens 限制

直接 curl 调 LLM API 时**必须**设置 `max_tokens`。没有这个，64KB transcript 跑 8+ 分钟没结果。

| max_tokens | 期望 Part B 完整度 | 推荐场景 |
|---|---|---|
| 2000 | 完整 | 30 分钟播客 |
| **4000** ⭐ | 完整 | 60 分钟播客（默认） |
| 6000 | 完整 | < 8K chars 输入 |

**实测**：max_tokens=4000 + 30K chars 输入 = ~2 分钟完成。
